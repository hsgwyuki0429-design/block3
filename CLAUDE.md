# block3 プロジェクト運用メモ

## 自動マージ方針(重要)

このリポジトリのオーナーの指示により、**コードを更新したら自動的に `main` へマージする**。
どのセッションでも、毎回オーナーに確認を取らずに以下を実行してよい:

1. 変更を作業ブランチにコミットして push する。
2. `main` への Pull Request を作成する。
3. その Pull Request を `main` にマージする(merge コミット方式)。

- この方針はオーナーによる恒久的な事前承認である(2026-07-23 に指示)。マージ前に確認を取り直す必要はない。
- ただしコードは push/PR 前に、可能な範囲で動作確認(構文チェック・実機/ヘッドレスでの読み込み確認など)を行い、壊れた状態をマージしないこと。
- マージ後は作業ブランチを最新の `main` から作り直してから次の作業に入る。

## プロジェクト概要

- `index.html` … 単一ファイルで完結する 3D ブロックパズル「ブロックブラスト3D」(three.js r128)。
- `manifest.json` / `icons/` … PWA(ホーム画面追加・全画面表示)対応とアプリアイコン。
